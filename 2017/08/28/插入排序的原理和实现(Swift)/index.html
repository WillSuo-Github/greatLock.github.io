插入排序的原理和实现(Swift)

发表于 2017-08-28 | 浏览

插入排序

原理

插入排序的原理就像下图显示的一样:

排序

就是遍历整个数组, 让取出的当前元素A和之前的所有元素对比, 由于对比的元素B是倒序取出的, 所以我们一旦我们发现 B>A 我们就将B的位置向右移动一位, 一直到B<A或者没有B元素可取了我们才停止比较并取出下一轮的A元素进行上述操作

实现

既然原理已经很清晰了我们来看一下代码的逐步实现, 首先给出一个乱序的数组:

var array = [6,5,3,1,8,7,2,4]

我们从角标1开始进行第一层的for循环

最先取出的元素是角标为i = 1的currentNum = 5与角标为0的元素6相比较, 元素6要比5大所以我们把元素6向右移动一位, 然后把原来是6的位置填充为5

得出的数组是

[5,6,3,1,8,7,2,4]

然后开始第一层的第二次循环, 当i= 2的时候currentNum = 3与它之前的元素6比较, 由于6>3我们把6向右移动一位, 然后把6的位置填充为3 , 再比较它之前的元素5, 5也大于3我们将5也向右移动一位, 然后把5的位置填充为3最后得到的数组应该是

[3,5,6,1,8,7,2,4]

以此类推我们的代码写出来应该长这个样子

下面我们就来分析一下这段代码的时间复杂度, 先假设array的长度是n, 每一行的执行代价为 Ci , 那么第二行的代码的执行代价为 C2 执行次数为n - 1, 第三行的执行代价为 C3 执行次数为n - 2 ……最终的整理:

image

上述中的最坏情况指的是array数组碰巧是一个倒序排列的数组, 使得第5行的判断每次都是true, 所以这段代码在最坏情况下的运行时间为:

T(n) = C2 * (n - 1) + C3 * (n - 2) + C4 * \(\sum_{i=1}^{n-1} i\) + C5 * \(\sum_{i=1}^{n-1} i\) + C6 * \(\sum_{i=1}^{n-1} i\) + C7 * \(\sum_{i=1}^{n-1} i\)

​ = C2 * (n - 1) + C3 * (n - 2) + C4 * (\(\frac{n * (n - 1)}{2}\) ) + C5 * (\(\frac{n * (n - 1)}{2}\) ) + C6 * (\(\frac{n * (n - 1)}{2}\) ) + C7 * (\(\frac{n * (n - 1)}{2}\) )

​ = (\(\frac{C₄}{2}\) + \(\frac{C₅}{2}\) + \(\frac{C₆}{2}\) + \(\frac{C₇}{2}\) ) $n^2 $ + (C2 + C3 - \(\frac{C₄}{2}\) - \(\frac{C₅}{2}\) - \(\frac{C₆}{2}\) - \(\frac{C₇}{2}\))n - (C2 + C3)

我们可以把时间简单的表示为 a\(n^2\) + bn + c , 而其中的常量a, b和c 又是依赖语句执行代价Ci. 因此它是n的二次函数也记为 \(\theta\)(\(n^2\)).

那么这段代码在最好的情况下是怎样的呢

image

T(n) = C2 * (n - 1) + C3 * (n - 2) + C4 * (n - 1) + C5 * (n - 1) + C9 * (n - 1)

​ = (C2 + C3 + C4 + C5 + C9) * n - (C2 + 2 * C3 + C4 + C5 + C9)

同理它是n的线性函数, 记为\(\theta\)(n).

总结

在实际项目运用中, 对于很多算法而言最坏情况是经常出现的, 比如说, 当数据库检索一条特定的信息的时候, 如果该信息不在数据库中存在, 检索算法的最坏情况就会出现, 在很多应用中, 对缺失信息的检索是很频繁的. 所以我们在讨论一个算法的复杂度的时候往往说的就是最坏的情况. So对于插入排序来讲它的时间复杂度就是\(\theta\)(\(n^2\)).
